ROLE: Senior frontend engineer with privacy/legal compliance awareness.

CONTEXT:
- Block 7 of 35. @kyc/ui is in place.
- Before building the marketing site we need compliance scaffolding: Privacy Policy, Terms of Service, Cookie Policy pages, and a GDPR-compliant cookie consent banner that the user can revisit later.

GOAL:
Add a `Legal` module to `@kyc/ui` exporting a reusable Cookie Consent system and three MDX-driven legal page templates that accept locale-aware content.

REQUIREMENTS:

1. Install in @kyc/ui: `@mdx-js/react`, `js-cookie`, `@types/js-cookie`.

2. `packages/ui/src/legal/CookieConsent.tsx` ("use client"):
   - Floating bottom banner on first visit. Premium dark styling, gold accent.
   - Three actions: "Accept all", "Reject non-essential", "Customize".
   - "Customize" opens a Drawer with toggleable categories: Necessary (always on, disabled toggle), Analytics, Marketing, Functional.
   - Persists choice in cookie `kyc_consent` (1-year expiry) and emits a `window.dispatchEvent(new CustomEvent('kyc:consent-changed', { detail }))`.
   - Hook `useCookieConsent()` returning `{ consent, hasDecided, update, reopen }`.
   - `CookieConsentProvider` wraps app, mounts the banner conditionally, exposes context.
   - Accessibility: focus trap, ESC to close drawer, role="dialog".

3. `packages/ui/src/legal/LegalPage.tsx` (Server Component compatible):
   - Pure presentational layout: container, prose styling tuned to dark+gold (custom `.prose-premium` class), table of contents (sticky on desktop), last-updated date.
   - Accepts `title`, `lastUpdated`, `tableOfContents[]`, `children`.

4. `packages/ui/src/legal/contents/`:
   - `privacy.en.mdx`, `privacy.ru.mdx`, `privacy.uk.mdx`
   - `terms.en.mdx`, `terms.ru.mdx`, `terms.uk.mdx`
   - `cookies.en.mdx`, `cookies.ru.mdx`, `cookies.uk.mdx`
   - Generic GDPR-compliant baseline content for an EU/CIS-targeted business club, with placeholders `[[COMPANY_NAME]]`, `[[CONTACT_EMAIL]]`, `[[JURISDICTION]]`, `[[EFFECTIVE_DATE]]`. Include sections: data we collect, lawful basis, retention, user rights (access, rectification, erasure, portability), cookies, third parties (list Resend, Supabase, UploadThing, PostHog, Sentry, Upstash, Vercel), contact, changes.
   - Export `getLegalContent(doc, locale)` helper that returns the MDX source string for runtime rendering by the apps.

5. `packages/ui/src/legal/index.ts` — barrel export.

6. Add Storybook story for `CookieConsent` (with "Reset cookie" control) and a `LegalPage` story rendered with the English Privacy MDX.

7. Update `packages/ui/README.md` with a "Legal & Consent" section explaining: how apps render legal pages (they'll create routes that import the MDX), how to extend categories, where to place region-specific addenda.

8. Document the consent integration contract: which scripts should be gated by which category (e.g., PostHog -> analytics, Web Push -> functional, future ads -> marketing).

ACCEPTANCE CRITERIA:
- Storybook shows the banner; clicking each option persists the right cookie value and dispatches the event.
- Drawer accessible via keyboard.
- All three MDX docs render through `LegalPage` without errors and respect the premium typography.
- Typecheck + lint green.

DON'Ts:
- Do not call a remote consent provider (no OneTrust, Cookiebot). Keep this self-hosted.
- Do not auto-load any analytics or tracking script from inside this module.
- Do not assume Next.js — keep components framework-agnostic.
- Do not store consent in localStorage only (must be a cookie so SSR/middleware can read it later).

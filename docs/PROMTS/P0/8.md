GOAL:
Build `@kyc/i18n` exporting message catalogs (RU/EN/UK), shared formatting helpers, and a typed message key system. Pull every visible string from the mockups into the catalogs.

REQUIREMENTS:

   - name `@kyc/i18n`, type "module"
   - deps: next-intl@latest
   - exports: `./messages/*`, `./config`, `./formatters`, `./types`

2. `packages/i18n/src/config.ts`:
   - `locales = ['ru', 'en', 'uk'] as const`
   - `defaultLocale = 'ru'`
   - `localeNames = { ru: 'Русский', en: 'English', uk: 'Українська' }`
   - `localeDirections = { ru: 'ltr', en: 'ltr', uk: 'ltr' }`

   - `common.json` — buttons, statuses, errors, generic words ("loading", "save", "cancel", "search", "language", "country", "city", "category", "all", "back", "next", "save", "close", "yes", "no").

   - `catalog.json` — list page, filters, sorting, empty states, partner detail page strings, reviews, ratings.
   - `auth.json` — login/register/forgot/magic-link copy.
   - `applications.json` — "Apply for a card" form labels, validation messages, success screen.
   - `referrals.json` — referral page copy, share text templates.
   - `events.json` — list, detail, registration states, capacity warnings.
   - `promotions.json` — hot deals widget, expiry countdown, code copy.
   - `chat.json` — empty inbox, typing, message statuses, operator names.
   - `notifications.json` — push subscribe prompts, in-app toasts.

   - `legal.json` — section titles for privacy/terms/cookies plus the cookie banner copy.

3. Provide complete RU translations (we'll iterate EN/UK keys but they must all exist — for EN/UK use real translations, not "TODO"). Be especially careful to match mockup wording exactly in RU.

4. `packages/i18n/src/formatters.ts`:
   - `formatPrice(value, currency, locale)`
   - `formatDiscount(percent, locale)` — "−25%" with correct sign and locale separator
   - `formatRelativeTime(date, locale)` — using Intl.RelativeTimeFormat
   - `formatMemberCardId(id)` — keeps "KYC-UA-000123" canonical
   - `formatPhone(value, country)` — basic E.164 formatting

5. `packages/i18n/src/types.ts`:
   - Generate `Messages` type from RU JSON shape so message keys are autocompleted.
   - Use `as const` JSON imports and a typed `getMessages(locale)` helper.

6. `packages/i18n/src/index.ts` — public barrel.

7. `packages/i18n/README.md`:
   - How to add a new namespace.
   - How to add a new locale.
   - Rule: "no string in app code — only message keys" (link to next-intl docs).
   - How RU mockup strings map to keys (small reference table).

ACCEPTANCE CRITERIA:
- All three locales have the same key shape (write a small `validate-locales.ts` script and a `pnpm test` for the package that fails if any locale is missing keys present in RU).
- `formatDiscount(25, 'ru')` returns "−25%".
- Importable from both apps via workspace.
- Typecheck green.

DON'Ts:
- Do not nest message keys more than 3 levels deep.
- Do not inline ICU plural forms incorrectly — use next-intl's `{count, plural, one {...} other {...}}` syntax with all three locale-specific rules (especially `one/few/many/other` for RU and UK).
- Do not put admin-only strings into member/marketing files (clear separation).
- Do not commit any locale-detection middleware here — that belongs to apps/web in Block 9.

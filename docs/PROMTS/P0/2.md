ROLE: Senior frontend platform engineer. Create the shared configuration package that all apps and packages will consume.

CONTEXT:
- Block 2 of 35. Monorepo from Block 1 is in place.
- This package centralizes TypeScript, Biome, and Tailwind v4 base configs so apps stay consistent.

GOAL:
Build `@kyc/config` exporting tsconfig presets, a Biome config, and a Tailwind v4 preset with the premium dark + gold design tokens.

REQUIREMENTS:

1. `packages/config/package.json`:
   - name `@kyc/config`, private, type "module"
   - exports map for `./tsconfig/base.json`, `./tsconfig/nextjs.json`, `./tsconfig/react-library.json`, `./biome.json`, `./tailwind-preset.ts`
   - peerDeps: typescript, tailwindcss@^4
   - devDeps: @biomejs/biome@latest

2. TypeScript configs (`packages/config/tsconfig/*.json`):
   - `base.json`: strict, target ES2022, moduleResolution Bundler, skipLibCheck, isolatedModules, noUncheckedIndexedAccess, exactOptionalPropertyTypes, verbatimModuleSyntax, jsx preserve.
   - `nextjs.json`: extends base, includes Next.js types, plugin "next".
   - `react-library.json`: extends base, jsx react-jsx, declaration true, composite true.

3. `packages/config/biome.json`:
   - formatter: spaces, width 100, semi always, single quotes for JS, double for JSX.
   - linter: recommended + a11y + nursery rules. Severity error for `noUnusedImports`, `noUnusedVariables`, `useExhaustiveDependencies`.
   - ignore: `**/node_modules`, `**/.next`, `**/dist`, `**/.turbo`, `**/prisma/migrations`.

4. `packages/config/tailwind-preset.ts` — Tailwind v4 preset (use the v4 config-via-CSS approach is fine, but here export a JS preset for compatibility):
   - Custom color palette (premium dark + gold from the mockups):
     - `gold`: 50 #FBF5E6, 100 #F5E6B8, 200 #EFD68A, 300 #E9C65C, 400 #E3B62E, 500 #D4AF37 (PRIMARY), 600 #B8932A, 700 #8C6E1F, 800 #604A14, 900 #34270A
     - `ink` (deep blacks for surfaces): 50 #F5F5F7, 100 #E5E5EA, 200 #C7C7CC, 300 #8E8E93, 400 #636366, 500 #48484A, 600 #3A3A3C, 700 #2C2C2E, 800 #1C1C1E, 900 #0E0E10, 950 #000000
     - semantic tokens via CSS vars: `--bg-surface`, `--bg-elevated`, `--text-primary`, `--text-muted`, `--accent`, `--accent-fg`, `--border`, `--success`, `--danger`
   - Custom font families: `display` (Cormorant Garamond, serif), `sans` (Manrope, system-ui).
   - Border radius scale, shadow scale with a `gold-glow` premium shadow `0 0 24px rgba(212, 175, 55, 0.35)`.
   - Container queries plugin enabled.
   - Dark mode: class strategy, default to dark.

5. Add JSDoc comments to the preset explaining each token's intent.

6. Update root `package.json` scripts: `format` runs biome format, `lint` runs biome lint via turbo (each package will reference biome.json from @kyc/config).

ACCEPTANCE CRITERIA:
- `pnpm --filter @kyc/config build` succeeds (or no-op if no build step needed).
- Other packages can import `@kyc/config/tsconfig/base.json` via "extends".
- `npx biome check --config-path packages/config/biome.json .` runs (will pass since no source yet).

DON'Ts:
- Do not add ESLint or Prettier — Biome only.
- Do not hardcode the palette anywhere other than this preset.

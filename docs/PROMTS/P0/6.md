ROLE: Senior UI engineer + designer. Build the shared component library used by both apps.

CONTEXT:
- Block 6 of 35. @kyc/config has the Tailwind preset with the premium dark+gold tokens.
- This package hosts every reusable visual component. apps/web and apps/admin both import from `@kyc/ui`. The premium aesthetic from the mockups must be preserved.

GOAL:
Bootstrap `@kyc/ui` with Tailwind v4, shadcn-style primitives, premium custom components, and a Storybook 8 instance for visual development.

REQUIREMENTS:

1. `packages/ui/package.json`:
   - name `@kyc/ui`, type "module"
   - exports: `./styles/globals.css`, `./components/*`, `./hooks/*`, `./lib/*`
   - deps: react@19, react-dom@19, tailwindcss@^4, @tailwindcss/postcss, tailwind-merge, clsx, class-variance-authority, @radix-ui/react-* (slot, dialog, sheet, select, tabs, toast, dropdown-menu, popover, tooltip, avatar, checkbox, radio-group, switch, scroll-area, separator, accordion, navigation-menu, progress, label, hover-card, aspect-ratio), lucide-react, framer-motion, sonner (toasts), cmdk (command palette), vaul (mobile drawer), react-aria-components (a11y bits).
   - devDeps: storybook@^8, @storybook/react-vite, @storybook/addon-essentials, @storybook/addon-a11y, vite.

2. `packages/ui/styles/globals.css` (Tailwind v4 CSS-first):
   - `@import "tailwindcss";`
   - `@theme inline` block exposing the palette + semantic tokens from @kyc/config.
   - CSS custom properties for semantic tokens (light + dark variants; dark default).
   - Base layer: html background `var(--bg-surface)`, text `var(--text-primary)`, font-family display + sans, smooth scroll, antialiased.
   - Custom utilities: `.gold-gradient` (linear gradient gold-400 → gold-600), `.text-gold-shine` (animated shine), `.premium-border` (1px gold-500/30), `.bg-elevated`.

3. `packages/ui/src/lib/cn.ts` — `cn(...inputs)` using clsx + tailwind-merge.

4. `packages/ui/src/components/` — premium implementations of shadcn primitives. Each file is a Server-Component-safe export plus a "use client" wrapper where needed. Components to deliver:
   - Primitives: Button (with `variant`: default, gold, ghost, outline, destructive, link; `size`: sm, md, lg, icon), IconButton, Input, Textarea, Select, Combobox, Checkbox, RadioGroup, Switch, Label, FormField (wraps RHF), Slider, Tabs, Accordion, Tooltip, Popover, DropdownMenu, NavigationMenu, Dialog, Sheet, Drawer (vaul), AlertDialog, Toast (Sonner wrapper), Badge, Avatar, Skeleton, Separator, Progress, ScrollArea, Card (Header/Title/Description/Content/Footer), AspectRatio, Command (cmdk), EmptyState, Spinner.
   - Custom premium components:
     - `GoldButton` — premium gold CTA with subtle inner glow + framer-motion hover scale.
     - `PremiumCard` — dark elevated surface with thin gold border, optional `glow` prop.
     - `MembershipCardSurface` — pure presentational dark+gold card surface used for the digital membership card (data fed in later block).
     - `CategoryIcon` — accepts a lucide icon name (string) and renders inside a gold-tinted rounded square.
     - `CountryFlag` — emoji or SVG flag with name (renders the "США / Канада / ..." chips from the mockup).
     - `BottomNav` — fixed mobile bottom nav with 5 slots, active state with gold underline + scale animation; accepts items[] of `{ icon, label, href, active }`.
     - `LocaleSwitcher` — dropdown for RU/EN/UK (no i18n logic here — pure UI; takes value + onChange).
     - `Logo` — wordmark "KYLYVNYK CLUB" using the display font with optional gold accent.
     - `PartnerCard` — used in catalog grid (logo, name, category, city, discount badge, favorite icon button).
     - `DiscountBadge` — green pill "Скидка для вас 25%".
     - `SearchBar` — input + leading lucide Search icon + variants (compact, hero).
     - `FilterPill` — selectable pill for category chips.
     - `StatTile` — for "Why choose us" / admin dashboard.
     - `SectionHeader` — gold rule + centered title used everywhere on the marketing page.
   - All custom components use CVA for variants and accept `className`.

5. `packages/ui/src/hooks/`: useMediaQuery, useIsClient, useCopyToClipboard, useDisclosure, useLocalStorage (SSR-safe).

6. Storybook in `packages/ui/.storybook/` (main.ts, preview.ts). Configure dark background by default. Add stories for: Button (all variants), GoldButton, PremiumCard, MembershipCardSurface, BottomNav, PartnerCard, LocaleSwitcher, SearchBar, SectionHeader, StatTile, EmptyState. Use realistic content from the mockups (Lex Pro, Account Expert, Megapolis, Fitness Life, etc.).

7. `packages/ui/README.md` documenting: how to import, how to add a new component, naming conventions, the difference between primitive vs custom.

8. tsconfig extends `@kyc/config/tsconfig/react-library.json`. Add `package.json` script `storybook` (storybook dev -p 6006) and `build-storybook`.

ACCEPTANCE CRITERIA:
- `pnpm --filter @kyc/ui storybook` opens Storybook on :6006 and all listed stories render without console errors.
- Typecheck and Biome lint pass.
- All custom premium components visually match the mockup palette (dark backgrounds, gold accents).
- No component pulls in app-level concerns (no Next.js Link/Image imports — use plain `<a>` and `<img>`; the apps will adapt).

DON'Ts:

- Do not hardcode colors — always use Tailwind classes referencing the palette tokens.

- Do not introduce a global state library here.

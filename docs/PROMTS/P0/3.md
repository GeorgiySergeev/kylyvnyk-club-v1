ROLE: Senior backend engineer. Design and implement the database layer.

CONTEXT:
- Block 3 of 35. Monorepo + config package are in place.
- Database: Supabase Postgres (we connect via the connection string only; we do NOT use Supabase Auth or Supabase Storage).
- ORM: Prisma. The schema must support all 35 MVP blocks: members, partners, applications, deals, referrals, favorites, events, promotions, chats, push subscriptions, audit logs, onboarding checklist.

GOAL:
Build `@kyc/db` containing the Prisma schema, a generated client wrapper, a seed script with realistic demo data, and helper scripts.

REQUIREMENTS:

1. `packages/db/package.json`:
   - name `@kyc/db`, type "module"
   - scripts: `db:generate` (prisma generate), `db:push` (prisma db push), `db:migrate` (prisma migrate dev), `db:deploy` (prisma migrate deploy), `db:seed` (tsx prisma/seed.ts), `db:studio` (prisma studio)
   - deps: @prisma/client@latest, prisma@latest, tsx, @faker-js/faker, bcryptjs, nanoid
   - tsconfig extends `@kyc/config/tsconfig/base.json`

2. `packages/db/prisma/schema.prisma`:
   - generator client: provider prisma-client-js, output to `../generated/client`
   - datasource db: provider postgresql, url env("DATABASE_URL"), directUrl env("DIRECT_URL") (Supabase pooler pattern)
   - Models (all with `id String @id @default(cuid())`, `createdAt`, `updatedAt`, `deletedAt DateTime?` where soft-delete makes sense):
     * **User** — id, email (unique), emailVerified, name, image, hashedPassword (nullable for OAuth), role (enum UserRole: MEMBER, ADMIN, SUPER_ADMIN), twoFactorSecret, twoFactorEnabled, locale, createdAt, updatedAt. Relations: accounts, sessions, member, adminAuditLogs.
     * **Account, Session, VerificationToken** — standard Auth.js v5 Prisma adapter models.
     * **Member** — userId (unique FK), memberCardId (unique, format "KYC-{COUNTRY}-{SEQ}"), countryId, cityId, phone, status (enum: ACTIVE, SUSPENDED, PENDING), referredById (self-relation), onboardingProgress (Json), createdAt, deletedAt.
     * **MembershipApplication** — name, email, phone, countryId, source, message, status (enum: NEW, CONTACTED, APPROVED, REJECTED), assignedToId (FK User admin), notes, createdAt.
     * **Country** — code (unique, ISO 3166-1 alpha-2), nameEn, nameRu, nameUk, flagEmoji, displayOrder, isActive.
     * **City** — countryId, nameEn, nameRu, nameUk, displayOrder. Unique on (countryId, nameEn).
     * **Category** — slug (unique), iconName (lucide icon key), nameEn, nameRu, nameUk, displayOrder, isActive.
     * **Partner** — slug (unique), nameEn, nameRu, nameUk, descriptionEn, descriptionRu, descriptionUk, logoUrl, coverUrl, cityId, address, latitude, longitude, phone, website, email, discountValue (Int, percent), discountText (string), status (enum: DRAFT, PUBLISHED, ARCHIVED), publishedAt, averageRating Float?, reviewCount Int @default(0), createdAt, updatedAt, deletedAt. Full-text search vector column via `@@index([...])` and a raw migration later.
     * **PartnerCategory** — partnerId, categoryId. M:N join.
     * **Favorite** — memberId, partnerId, createdAt. Unique on (memberId, partnerId).
     * **Deal** — memberId, partnerId, amount Decimal, currency, status (enum: PENDING, CONFIRMED, DISPUTED), happenedAt.
     * **Review** — dealId (unique), memberId, partnerId, rating Int 1-5, text, status (enum: PENDING, APPROVED, REJECTED), moderatedById (FK admin), moderatedAt, createdAt.
     * **Referral** — referrerId (FK Member), refereeId (FK Member nullable until signup), code, source, status (enum: PENDING, COMPLETED), reward Decimal default 0, createdAt.
     * **Event** — slug (unique), titleEn/Ru/Uk, descriptionEn/Ru/Uk, coverUrl, startsAt, endsAt, locationName, cityId, address, capacity Int?, status (enum: DRAFT, PUBLISHED, CANCELLED, COMPLETED).
     * **EventRegistration** — eventId, memberId, status (REGISTERED, CHECKED_IN, CANCELLED), checkedInAt. Unique on (eventId, memberId).
     * **Promotion** — partnerId, titleEn/Ru/Uk, descriptionEn/Ru/Uk, discountValue, code, startsAt, endsAt, usageLimit Int?, usageCount Int default 0, status (enum: DRAFT, ACTIVE, PAUSED, EXPIRED).
     * **Conversation** — memberId, assignedAdminId (FK User), status (OPEN, PENDING, CLOSED), lastMessageAt, unreadByMember Int default 0, unreadByAdmin Int default 0.
     * **Message** — conversationId, senderId (FK User), body, attachments (Json), readAt, createdAt.
     * **PushSubscription** — userId, endpoint (unique), p256dh, auth, userAgent, createdAt.
     * **PushCampaign** — title, body, url, audience (Json filter), scheduledAt, sentAt, status (DRAFT, SCHEDULED, SENT, FAILED), sentCount Int default 0, createdById.
     * **OnboardingStep** — key (unique), titleEn/Ru/Uk, descriptionEn/Ru/Uk, ctaHref, displayOrder, isActive.
     * **AuditLog** — actorId (FK User), action (string), entity (string), entityId, beforeJson Json?, afterJson Json?, ip, userAgent, createdAt @default(now()) @db.Timestamptz.
   - Add explicit `@@index` on FK columns, slug fields, status fields, and date fields used in queries.

3. `packages/db/src/index.ts`:
   - Export a singleton `prisma` instance (avoids client multiplicity in Next.js dev) using `globalThis` pattern.
   - Re-export all generated types and enums.

4. `packages/db/prisma/seed.ts`:
   - Truncate everything in dev only.
   - Seed: 5 countries (US, CA, CH, UA, MD), 3-4 cities per country, 12 categories matching the mockup (Legal, Accounting, Real Estate, Beauty, Fitness, Restaurants, Transport, IT & Marketing, Medicine, Online Services, Education, Travel).
   - 30 demo partners using faker (assign 1-2 categories each, realistic discount 5-30%, status PUBLISHED).
   - 2 admin users: super@kylyvnyk.club (password "ChangeMe123!") and admin@kylyvnyk.club. Use bcryptjs to hash.
   - 5 demo members with memberCardIds like "KYC-UA-000001".
   - 4 onboarding steps from blueprint.
   - 3 demo events, 5 demo promotions.

5. `packages/db/.env.example` with DATABASE_URL and DIRECT_URL placeholders pointing to Supabase pooler + direct URLs (with comments explaining each).

6. `packages/db/README.md` with: setup steps (create Supabase project, copy connection strings, run db:push then db:seed), how to extend the schema, soft-delete convention.

ACCEPTANCE CRITERIA:
- `pnpm --filter @kyc/db db:generate` produces a working Prisma client.
- With a valid Supabase DATABASE_URL: `pnpm --filter @kyc/db db:push && pnpm --filter @kyc/db db:seed` populates demo data.
- `pnpm --filter @kyc/db db:studio` opens Prisma Studio with the expected tables.
- All FKs and unique constraints are in place.

DON'Ts:
- Do NOT use Supabase JS client. Connect to Postgres via Prisma only.
- Do NOT hash passwords with anything other than bcryptjs (cost 12).
- Do NOT inline any business logic in seed — keep it data-only.
- Do NOT delete the migrations folder once created — but for this initial setup use `db:push` (we'll switch to migrations in CI block).

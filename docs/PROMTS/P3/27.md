1. **Implement procedures** under `packages/api/src/routers/admin/`:
   - **applications.ts**:
     - `list`: { cursor?, limit, status?: 'all'|'NEW'|'CONTACTED'|'APPROVED'|'REJECTED', search?, countryId?, sort?: 'newest'|'oldest' }.
     - `get`: full row with referral info.
     - `assign`: { id, adminUserId|null } — sets MembershipApplication.assignedToId.
     - `setStatus`: { id, status: 'NEW'|'CONTACTED'|'REJECTED' } — only allows manual transitions (NOT APPROVED — that goes through approve()).
     - `addNote`: { id, note (1-1000) } — appends to MembershipApplication.notes (newline-separated, with timestamp + author).
     - `approve` (auditedAdminProcedure):
       - Input { id, locale?: override applicant locale }.
       - Steps in a single Prisma transaction:
         1. Fetch application; reject if already APPROVED.
         2. Either find existing User by email or create one:
            - If exists: must have role=MEMBER (else reject — "Email already used by admin"). Reuse user.
            - If absent: create User { email, name, role: 'MEMBER', locale, image: null, hashedPassword: null } (user will set password via magic link).
         3. Generate memberCardId:
            - Format `KYC-{CC}-{NNNNNN}` where CC = applicant country code, NNNNNN = zero-padded next sequence per-country.
            - Maintain a `MemberSequence { countryCode @id, lastValue Int }` table. Add migration in @kyc/db. Atomic increment in transaction.
         4. Create Member (or update existing PENDING one) with status=ACTIVE, memberCardId, countryId, cityId nullable, phone from app.
         5. If app.referralCode present, find Referral row and set status=COMPLETED, refereeId=newMember.id; call `markOnboardingStep(referrerMemberId, 'invite_friend')` helper.
         6. Set application.status='APPROVED', approvedAt=now, approvedById=actor.id.
         7. AuditLog 'application.approve' with full before/after.
       - After commit (outside transaction): fire-and-forget Resend email `templates/welcome-activate.tsx` to applicant — branded, includes magic-link button (triggers Auth.js Resend provider sign-in with callbackUrl=`/[locale]/dashboard`) and the assigned memberCardId.
       - Returns `{ memberId, memberCardId }`.
     - `reject`: { id, reason } — sets REJECTED + reason in notes; sends a polite rejection email (template `application-rejected.tsx`).
   - **members.ts**:
     - `list`: { cursor?, limit, search?, status?: 'all'|'ACTIVE'|'SUSPENDED'|'PENDING'|'DELETED', countryId?, sort?: 'newest'|'name'|'cardId' }.
     - `get`: full profile including counts (deals, reviews, favorites, referrals invited/completed) and recent activity (last 10 sign-ins from AuditLog).
     - `setStatus`: { id, status, reason? } — admin sets ACTIVE/SUSPENDED.
     - `softDelete`: { id, reason } — sets deletedAt on Member and User, revokes Session rows, fires email "Account suspended".
     - `restore`: { id }.
     - `resendMagicLink`: { id } — triggers Resend magic link for the linked email. Rate-limited 3/h/admin.
     - `forceLogoutAll`: { id } — deletes all Session rows for the user.
     - `regenerateCardId` (superAdminProcedure): rotates memberCardId — increments sequence; old card token invalidated by the JWT secret unchanged (token sub becomes stale). Audit-logged. Sends email.

4. **Auth.js v5 (admin variant)** in `apps/admin/src/server/auth/config.ts`:
   - Same Prisma adapter against same User table.
   - Providers: Credentials ONLY (no Google, no magic link — admins must own their password + TOTP).
   - Credentials flow has TWO steps:
     - **Step 1: email+password** — if password valid AND user.twoFactorEnabled, do NOT create session yet; instead set a short-lived signed cookie `kyc_admin_2fa` = JWT with `{ userId, expires: now+5m }`. Redirect to `/two-factor`.
     - **Step 2: TOTP code** — `/two-factor` page reads the cookie, accepts 6-digit code, calls `auth.adminVerify2FA` mutation; on success the server completes sign-in (creates full Session row) by setting the Auth.js session cookie via a server-action route. Clear `kyc_admin_2fa`.
   - Role gate in signIn callback: throw `AccessDenied` if role NOT IN ('ADMIN','SUPER_ADMIN').
   - Session callback: attach role, twoFactorEnabled, locale.
   - Session strategy: JWT (Edge-friendly middleware).
   - Pages: signIn `/login`, error `/login?error=`.

5. **TOTP implementation** in `apps/admin/src/server/auth/totp.ts`:
   - Use `otplib` (`authenticator`).
   - Issuer: "Kylyvnyk Admin". Account name: user.email.
   - Helpers: `generateSecret()`, `buildOtpAuthUrl(email, secret)`, `verifyCode(secret, code)` (with 1-step window).
   - Secret stored on User.twoFactorSecret (already in schema), encrypted at rest via AES-256-GCM using `ADMIN_TOTP_ENCRYPTION_KEY` env (32 bytes base64). Add `lib/crypto.ts` with `encrypt(text)` and `decrypt(payload)` (iv + tag + ciphertext format).

6. **Recovery codes** in `apps/admin/src/server/auth/recovery-codes.ts`:
   - On enabling 2FA, generate 10 single-use recovery codes (8 chars groups of 4, e.g., `AB12-CD34`).
   - Store hashed (bcrypt) in a new table `AdminRecoveryCode { id, userId, codeHash, usedAt? }`. Add migration in @kyc/db.
   - `validateRecoveryCode(userId, code)` returns true and marks usedAt.

7. **Implement admin auth procedures** in @kyc/api router `auth` (extending Block 10):
   - `auth.adminBegin2FASetup` (adminProcedure, but allowed for any role ≥ ADMIN even if not yet enabled): returns `{ secret (plain — only during setup), otpAuthUrl, qrPngDataUrl }`. Does NOT persist secret yet — caller must verify.
   - `auth.adminConfirm2FASetup` (adminProcedure): input { secret, code }. Verifies code against secret. On success: persist encrypted secret to User.twoFactorSecret, set twoFactorEnabled=true, generate + return 10 recovery codes (plain, one-time). Audit-logged.
   - `auth.adminVerify2FA` (publicProcedure, special — accepts the kyc_admin_2fa cookie as identity): input { code OR recoveryCode }. Verifies; on success the Route Handler proceeds to create session (see step 4 above). Audit-logged action 'admin.2fa.verify'.
   - `auth.adminDisable2FA` (adminProcedure): input { currentPassword, code }. Verifies password+TOTP, then clears secret + codes. Audit-logged. Restricted: SUPER_ADMIN cannot disable their own 2FA without a second SUPER_ADMIN approving (out of MVP scope — for now just allow; add TODO).
   - `auth.adminRegenerateRecoveryCodes` (adminProcedure): input { code }. Verifies TOTP, rotates codes (invalidates old).

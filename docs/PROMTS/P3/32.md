ROLE: Senior frontend engineer + data UX designer.

CONTEXT:
- Block 32 of 35 (final admin block). The dashboard is the first thing operators see after login. It should answer at a glance: how the club is growing, what needs attention now, and where to jump to act.

GOAL:
Build the admin dashboard with KPI tiles, time-series charts (recharts), action queues ("Applications to review", "Pending reviews", "Open conversations"), and a global `Cmd+K` command palette.

REQUIREMENTS:

1. **Implement procedures** in `packages/api/src/routers/admin/dashboard.ts`:
   - `overview`: returns aggregated metrics in a single call (denormalized for the dashboard):
     - members: { total, activeLast30d, newLast7d, newLast30d, suspendedCount }
     - applications: { newCount, contactedCount, approvedLast30d, rejectedLast30d, avgTimeToApprovalSec }
     - partners: { publishedCount, draftCount, archivedCount }
     - deals: { totalConfirmedLast30d, totalAmountLast30dByCurrency: { USD: x, EUR: x, ... } }
     - reviews: { pendingCount, approvedLast30d, avgRating }
     - chat: { openConversations, unassignedConversations, avgFirstResponseMinutesLast7d }
     - events: { upcomingCount, registrationsLast30d, checkInRateLast30d }
     - promotions: { activeCount, expiringIn7d }
     - referrals: { completedLast30d }
     - operators: { onlineCount }  // best-effort via presence — pass 0 in MVP
   - `timeSeries`: { metric: 'newMembers'|'applications'|'deals'|'reviews'|'pushDelivered', window: '7d'|'30d'|'90d', granularity: 'day'|'week' } → array of { bucket: Date, value: number }.
   - `recentActivity`: { limit default 10 } — most recent AuditLog rows enriched with actor info, formatted for the activity feed.
   - `actionQueues`: returns counts only (Applications NEW, Reviews PENDING, Conversations OPEN unassigned, Events starting in 24h) — same as `overview` but isolated for a smaller heartbeat call.

2. **Dashboard page** `apps/admin/src/app/[locale]/(admin)/page.tsx`:
   - Server-fetch overview in parallel with first chart series + recentActivity.
   - Sections:
     1. **Top KPI tiles row** (4 columns desktop / 2 mobile): Активные участники / Заявки в обработке / Партнёры / Открытые чаты. Each tile uses StatTile (Block 6) with delta vs previous period (computed in `overview`).
     2. **Action queues** card with 4 sub-items linking to the relevant page filtered by status: "{N} новых заявок", "{N} отзывов на модерации", "{N} неназначенных чатов", "{N} событий начнётся за 24 часа". Each row clickable.
     3. **Charts row** (2-up on desktop, stacked on mobile):
        - "Новые участники за 30 дней" — bar chart (recharts BarChart). Gold bars on slate bg. Tooltip with date + count.
        - "Заявки за 30 дней" — line chart with two series: new vs approved.
        - Toggle window: 7d / 30d / 90d (URL param + client-side switch refetches `timeSeries`).
     4. **Deals revenue card**: stacked bar chart per currency over the last 30d (totalAmountLast30dByCurrency). Convert nothing — show original currencies side by side.
     5. **Chat performance card**: KPI (avgFirstResponseMinutesLast7d) + tiny sparkline of daily averages. If avgFirstResponseMinutes > 30, the tile is highlighted red.
     6. **Recent activity feed**: timeline list from `recentActivity` — actor + action + entity + relative time + "Open" link to the AuditLog row.
     7. **Right rail (desktop only)**: "Шорткаты" — quick links to Создать партнёра, Создать событие, Создать промокод, Открыть инбокс. Each is a small premium card.
   - Auto-refresh: `actionQueues` polled every 30s via TanStack Query; full overview + charts refetch every 5 min on window focus.

3. **Charts setup** (recharts):
   - `apps/admin/src/components/charts/BarChart30d.tsx`, `LineChart30d.tsx`, `RevenueByCurrencyChart.tsx`, `Sparkline.tsx`. Each accepts data + locale formatting.
   - Theme palette pulled from CSS vars; render in a `<ChartContainer>` wrapper that handles ResizeObserver and aspect ratio.

4. **Command palette** (`Cmd+K`):

   - Actions: "Перейти в Партнёры", "Перейти в Заявки", "Перейти в Чат", "Создать партнёра", "Создать событие", "Создать промокод", "Найти участника по ID/email" (sub-prompt → opens `/members?search=...`), "Найти партнёра по slug/имени", "Открыть Audit Log", "Выйти".

DON'Ts:
- Do not let CI pass with skipped tests — fail-fast on any skipped suite unless flagged with `// reason:`.
- Do not deploy to production if `db:deploy` fails — keep migrate→deploy strictly sequential.
- Do not commit any .env file (even encrypted) — secrets live only in Vercel + GitHub Actions secrets.
- Do not enable Vercel's auto-deploy-on-push for production — let release.yml drive it after migrations.
- Do not let preview deploys hit the production DB — point preview env vars to a staging DATABASE_URL.
- Do not run cron jobs without CRON_SECRET enforcement.
- Do not skip CSP — set it strict and iterate if anything breaks.

   - Env: UPLOADTHING_TOKEN (new SDK uses single token). Update .env.example + env.ts.

2. **Web Push setup**:
   - Install: `web-push` (server), nothing extra on client.
   - Env: VAPID_PUBLIC_KEY, VAPID_PRIVATE_KEY, VAPID_SUBJECT (mailto:...). Update env.ts (use `NEXT_PUBLIC_VAPID_PUBLIC_KEY` for client-exposed key).
   - Helper `apps/web/src/server/push/webPush.ts`:
     - Lazy `webpush.setVapidDetails(...)`.
     - `sendPushTo(subscription, payload)` returns success/failure and prunes on 404/410 (delete row).
     - `sendPushToUser(userId, payload)` fetches all PushSubscription rows for the user, sends in parallel, prunes invalid.
   - Service worker integration: extend the existing Serwist SW file (from Block 9) to handle `push` and `notificationclick` events:
     - `self.addEventListener('push', ...)` — parses payload `{ title, body, url, tag, icon, badge }`, calls `registration.showNotification`.
     - `self.addEventListener('notificationclick', ...)` — focuses or opens the URL.
   - Make sure SW handler files compile under TypeScript with appropriate `lib: ['WebWorker']`.

3. **Implement procedures**:
   - `push.subscribe` (memberProcedure, rate-limited via genericPublicLimiter):
     - Input { endpoint, p256dh, auth, userAgent, channels: { chat: boolean, applications: boolean, promotions: boolean, events: boolean } (default all true) }.
     - Upserts a `PushSubscription` row by endpoint, attached to userId.
     - Stores channel preferences in a new column `channels Json` on PushSubscription (add migration).
     - Returns `{ ok: true }`.
   - `push.unsubscribe` (memberProcedure): input { endpoint } → delete row.
   - `push.updateChannels` (memberProcedure): input { endpoint, channels } → patch row.
   - `auth.changePassword` (memberProcedure, authLimiter):
     - Input { currentPassword, newPassword (same policy as register) }.
     - Verifies current password; rotates hash. Rejects if user has no password (OAuth-only — return BAD_REQUEST "Set initial password via magic link").
   - `auth.listSessions` (memberProcedure): returns current user's `Session` rows (from Auth.js Prisma adapter) with userAgent + ip + lastActive (we don't track ip — show userAgent and expires).
   - `auth.revokeSession` (memberProcedure): input { sessionToken } → delete row (NOT current session).
   - `member.requestAccountDeletion` (memberProcedure):
     - Soft-delete: set Member.deletedAt and User.deletedAt (add nullable column to User if missing), revoke all sessions, send admin email "Account deletion requested". Member stays for 30 days for recovery, then a cron (out of MVP) hard-deletes. For MVP, deletion request just freezes the account and notifies admin.

CONTEXT:
- Block 19 of 35. Dashboard has a tile "История сделок" → /(member)/deals. Reviews are read-only on the public partner page; here members write them.
- A Deal represents a confirmed transaction with a partner (created manually by admin in Block 27 or self-reported by member with PENDING status). MVP-friendly: members can self-report deals (PENDING), and they can leave a review only on a CONFIRMED deal. Admin confirms deals in Block 27.
- Reviews go through PENDING → APPROVED|REJECTED moderation (admin in Block 31). Members see their own reviews regardless of status, but with a status badge.

GOAL:
Implement `deals.list`, `deals.selfReport`, `deals.get`, `reviews.create`, build /(member)/deals page with deal list + review dialog, and the moderation-aware status surface.

REQUIREMENTS:

1. **Schema sanity check**:
   - Confirm @kyc/db has Deal { status: PENDING|CONFIRMED|DISPUTED, amount Decimal, currency, happenedAt, memberId, partnerId } and Review { dealId unique, status: PENDING|APPROVED|REJECTED, rating 1-5, text, moderatedById, moderatedAt }. Already in Phase 0 schema.
   - Add to seed: ~10 deals across the 5 seeded members in various statuses; 3 of them have approved reviews (already in Block 14), some are PENDING with no review yet, some CONFIRMED without a review.

2. **Implement procedures**:
   - `deals.list` (memberProcedure):
     - Input { cursor?, limit default 20, max 60, status?: 'all'|'pending'|'confirmed'|'disputed', withReview?: 'any'|'yes'|'no' }.
     - Returns own deals joined with partner (logo, name localized) and the associated review if any.
     - Sort: happenedAt desc.
     - Cursor pagination.
   - `deals.get` (memberProcedure): single deal by id, must belong to caller.
   - `deals.selfReport` (memberProcedure, rate-limited 10/hour/user via genericPublicLimiter):
     - Input { partnerId, amount (Decimal, min 0), currency (enum: USD|EUR|UAH|MDL|CHF|CAD), happenedAt (ISO ≤ now, ≥ now - 90 days), note (max 500, optional) }.
     - Inserts Deal { status: 'PENDING', memberId: session.member.id }.
     - Returns the new deal. Audit-logged.
   - `reviews.create` (memberProcedure, rate-limited via reviewLimiter):
     - Input { dealId, rating 1-5, text (10–2000 chars) }.
     - Validates: deal belongs to caller, deal.status === 'CONFIRMED', no existing review on that deal.
     - Inserts Review { status: 'PENDING' }.
     - Does NOT immediately update Partner.averageRating; aggregation runs on admin approval (Block 31).
     - Returns the review (with PENDING status visible to the author).
     - Audit-logged. Marks onboarding step? No — we don't have one for reviews. Skip.

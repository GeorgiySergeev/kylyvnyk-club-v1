DON'Ts:
- Do not store the consent checkbox value separately from the row — the row's presence implies consent (but record consentAt timestamp via createdAt).
- Do not block the response on email delivery — fire-and-forget with try/catch + console error.
- Do not expose application IDs in URL fragments that could be guessed/enumerated (the thanks page uses an opaque token query param, not the cuid — generate a one-time `?token=...` if you want; otherwise omit entirely and rely on session-less thanks page).
- Do not couple this form to auth — applicants are anonymous.

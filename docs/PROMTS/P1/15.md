GOAL:
Implement public read endpoints for promotions, add a "Hot Deals" section to the marketing homepage and a "Active promotions" rail to the catalog page, and ensure SSR + expiration handling.

REQUIREMENTS:

1. **Implement promotions read endpoints** in `packages/api/src/routers/promotions.ts`:
   - `promotions.listActive` (publicProcedure):
     - Input { limit default 12, max 24, partnerId?, categoryId?, countryId?, locale }
     - Returns active promotions: `status = 'ACTIVE'`, `startsAt <= now <= endsAt`, `usageLimit IS NULL OR usageCount < usageLimit`.
     - Joins partner (with partner.status='PUBLISHED', not soft-deleted), sorted by endsAt asc (most-expiring first).
     - Output items include: id, title (localized), description (localized), discountValue, code, startsAt, endsAt, usageRemaining (or null if unlimited), partner: { id, slug, name (localized), logoUrl, city (localized) }.
     - Cache: 60s revalidate.
   - `promotions.byPartner` (publicProcedure): list active promotions for a single partner, used on the detail page footer (small add).

   - Above "Reviews" section, if `promotions.byPartner.length > 0`, render a "Активные акции у этого партнёра" mini-section with up to 3 `PromotionCard`s.

   - PremiumCard with red/gold "limited" accent (use existing palette — no new colors).
   - Slots: partner logo small (top-right), title, short description (2 lines), big discount value, code (copy-to-clipboard chip), countdown to endsAt (live ticker — client island), "Перейти к партнёру" link.

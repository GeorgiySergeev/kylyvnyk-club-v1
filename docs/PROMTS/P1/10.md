# ROLE: Senior auth engineer

CONTEXT:
- Block 10 of 35. apps/web boots with placeholder pages and a working tRPC route.
- We use Auth.js v5 with Prisma Adapter against our Supabase Postgres. Auth.js owns sessions; tRPC reads the session from cookies via auth() in the Route Handler.
- Required flows for MVP: Credentials (email + password), Google OAuth, Magic Link via Resend, basic role gating (MEMBER / ADMIN / SUPER_ADMIN). 2FA enrollment lives in Block 24 (admin) — here we only stub the field plumbing.

GOAL:
Wire up Auth.js v5 in apps/web, expose authenticated context to tRPC, build /login, /register, /forgot, /verify-request, /error pages with full i18n, and protect /[locale]/(member)/* with middleware.

REQUIREMENTS:

1. Install in apps/web: next-auth@5.x (or beta if stable not yet released), @auth/prisma-adapter, resend, react-hook-form, @hookform/resolvers, zod, bcryptjs.

2. `apps/web/src/server/auth/config.ts`:
   - Export `authConfig` (the slim, Edge-safe config) and `auth, signIn, signOut, handlers` from `NextAuth(...)`.
   - Adapter: PrismaAdapter(prisma) from @kyc/db.
   - Strategy: JWT (so middleware can read it on the Edge without DB).
   - Session callback: attach `user.id`, `user.role`, `user.locale` to the session.
   - JWT callback: refresh role/locale from DB on sign-in only.
   - Providers:
     - Credentials: validates email + password against User.hashedPassword via bcryptjs. Throws CredentialsSignin on mismatch.
     - Google: clientId + clientSecret from env, scopes `openid email profile`.
     - Resend (built-in EmailProvider with Resend transport): sends magic link branded with KYLYVNYK CLUB (HTML template imported from `src/server/email/templates/magic-link.tsx` — use @react-email/components if available, otherwise plain HTML).
   - Pages: signIn '/[locale]/login', verifyRequest '/[locale]/verify-request', error '/[locale]/auth-error'.
   - Cookies: secure in prod, sameSite lax.
   - Events: `signIn` writes a non-blocking AuditLog row (`action: 'auth.signIn'`).

3. `apps/web/src/app/api/auth/[...nextauth]/route.ts` — export GET, POST from handlers.

4. Update `apps/web/src/middleware.ts`:
   - Combine next-intl middleware with Auth.js middleware in a single function:
     - For requests under `/[locale]/(member)`, call `auth()` (the Edge-safe one). If no session, redirect to `/[locale]/login?callbackUrl=...`.
     - Otherwise pass through next-intl.
   - matcher unchanged.

5. Update `apps/web/src/lib/trpc/context.ts`:
   - Call `auth()` from server config to retrieve session.
   - Build full context `{ session, prisma, ip: req.headers.get('x-forwarded-for'), userAgent: req.headers.get('user-agent') }` and pass into appRouter.

6. Implement these previously stubbed tRPC procedures in `packages/api/src/routers/auth.ts` (this is the first time we touch a router — the rest stay stubbed). Procedures:
   - `auth.register` (publicLimitedProcedure with authLimiter): input { email, password (min 10, must contain digit + letter), name, locale, consent: true, referralCode?: string }. Steps:
     1. Check user doesn't already exist.
     2. Hash password (bcrypt, cost 12).
     3. Create User { role: 'MEMBER' }.
     4. Create empty Member row linked to user (status: PENDING — full activation happens on application approval; this is the auth-only baseline).
     5. If `referralCode` provided, link Referral.
     6. Return `{ id, email }`.
   - `auth.requestMagicLink` (publicLimitedProcedure with authLimiter): input { email, callbackUrl }. Calls `signIn('resend', { email, redirect: false })` server-side. Always returns `{ ok: true }` (do not leak existence).
   - `auth.me` (publicProcedure): returns `ctx.session?.user ?? null`.
   - `auth.updateProfile` (memberProcedure): input partial { name, locale, image, phone }. Updates User and Member.

7. UI pages under `apps/web/src/app/[locale]/(auth)/`:
   - `layout.tsx` — centered card layout, premium dark+gold, Logo on top, locale switcher in corner.
   - `login/page.tsx`:
     - Tabs (from @kyc/ui Tabs): "Sign in with password" | "Magic link".
     - Password form (RHF + Zod, calls `signIn('credentials', ...)`).
     - Magic link form (single email field, calls tRPC `auth.requestMagicLink`).
     - Below: Google button (`signIn('google')`).
     - Link to /register, /forgot.
   - `register/page.tsx`: full registration form (name, email, password, confirm password, locale select, consent checkbox linking to /legal/terms and /legal/privacy, optional referralCode auto-filled from `?ref=`). On success, push to `/login?registered=1`.
   - `forgot/page.tsx`: single email field — sends a magic link with callbackUrl to a "set new password" page (use the same Resend transport; reuse `auth.requestMagicLink`).
   - `verify-request/page.tsx`: "Check your inbox" branded screen.
   - `auth-error/page.tsx`: maps Auth.js error codes to localized messages.

8. `apps/web/src/server/email/`:
   - `resend.ts` — Resend client singleton (env: RESEND_API_KEY, RESEND_FROM).
   - `templates/magic-link.tsx` — branded HTML email (dark/gold inline styles, fallback to plain text).
   - `templates/welcome.tsx` (rendered on `auth.register` success — fire-and-forget).
   - Helper `sendEmail({ to, subject, template, props })` with try/catch + console error.

9. Localization: all strings via @kyc/i18n `auth.json` namespace (already present from Block 8). Use `useTranslations('auth')` on client, `getTranslations` on server.

10. Update `auth.json` only if a key is genuinely missing — otherwise reuse existing keys.

11. `apps/web/.env.example` — append: NEXTAUTH_SECRET (32+ chars), NEXTAUTH_URL, AUTH_TRUST_HOST=true, GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, RESEND_API_KEY, RESEND_FROM ("Kylyvnyk Club <noreply@yourdomain>").

12. `apps/web/README.md` — append an "Authentication" section: providers, how middleware decides protection, how to add a new role check.

13. Vitest tests in `packages/api/src/__tests__/auth.test.ts`:

ACCEPTANCE CRITERIA:
- Register a user via /ru/register → user + member rows created with bcrypt hash.
- Sign in via credentials → /ru works, /ru/dashboard (placeholder still 404 — that's fine for now; create a temporary `/ru/(member)/dashboard/page.tsx` that just says "Member area").
- Visiting `/ru/(member)/dashboard` while logged out → redirect to /ru/login?callbackUrl=...
- Magic link click in inbox completes sign-in and lands on callbackUrl.
- Sign out clears session and routes to /ru.
- AuditLog row appears on each sign-in.
- All auth pages localized in RU/EN/UK.
- Vitest + typecheck + lint green.

DON'Ts:
- Do not use Supabase Auth in any form. Auth.js is the sole authority.
- Do not commit secrets — .env.example only.
- Do not allow the Credentials provider to leak whether an email exists (always return the same error message for unknown email vs wrong password).
- Do not put auth() calls in Client Components; use `useSession()` only.
- Do not skip role plumbing — even though only MEMBER is exercised here, the session callback must expose `role`.

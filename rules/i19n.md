# Internationalization Rules

## The Golden Rule

**No user-facing string may appear in source code.**
Every string → message key → `.json` file → `useTranslations` / `getTranslations`.

## Usage Patterns

### Server Components (App Router)

```tsx
import { getTranslations } from "next-intl/server";

export default async function Page({ params: { locale } }) {
  const t = await getTranslations({ locale, namespace: "catalog" });
  return <h1>{t("title")}</h1>;
}
```

### Client Components

```tsx
"use client";
import { useTranslations } from "next-intl";

export function SearchBar() {
  const t = useTranslations("catalog");
  return <input placeholder={t("searchPlaceholder")} />;
}
```

### tRPC Errors Returned to UI

```ts
// In procedure:
throw new TRPCError({
  code: "NOT_FOUND",
  message: "partner_not_found", // ← message KEY, not text
});

// In client error handler:
const t = useTranslations("errors");
toast.error(t(error.message));
```

## Namespace Assignment

Do not mix concerns within a namespace:

| Namespace     | Contents                                          |
| ------------- | ------------------------------------------------- |
| common        | Shared: save, cancel, loading, yes, no, back      |
| marketing     | Landing page, hero, CTAs, how-it-works section    |
| member        | Dashboard, card, bottom-nav labels                |
| catalog       | Filters, sort, pagination, partner card labels    |
| auth          | Login/register forms, errors, magic-link copy     |
| applications  | Apply form, validation messages, success screen   |
| referrals     | Referral page, share templates, FAQ               |
| events        | List, detail, registration states                 |
| promotions    | Hot deals, expiry, code copy                      |
| chat          | Widget, inbox, canned responses                   |
| notifications | Push prompts, in-app toasts                       |
| onboarding    | Checklist titles, step CTAs                       |
| legal         | Privacy/terms/cookies section titles, banner copy |
| admin         | All admin UI strings                              |
| errors        | tRPC error codes mapped to user messages          |

## Adding New Messages

When adding a new string, YOU MUST:

1. Add the key to `packages/i18n/messages/ru/{namespace}.json`
2. Add the key to `packages/i18n/messages/en/{namespace}.json`
3. Add the key to `packages/i18n/messages/uk/{namespace}.json`
4. Run `pnpm --filter @kyc/i18n test` to verify key parity

## Key Naming Conventions

```jsonc
// ✅ camelCase, grouped by feature, max 3 levels deep
{
  "filter": {
    "country": "Страна",
    "city": "Город",
    "category": "Категория",
  },
}
// ❌ No snake_case, no PascalCase, no >3 levels
```

## Pluralization — Use ICU Plural Syntax

```jsonc
// ✅ Correct (Russian has one/few/many/other)
{
  "partnerCount": "{count, plural, one {# партнёр} few {# партнёра} many {# партнёров} other {# партнёра}}",
}
// ❌ Never "partnerCount1", "partnerCount2_4", etc.
```

## Date / Number Formatting

Always use `@kyc/i18n` formatters:

```ts
import { formatDiscount, formatRelativeTime } from "@kyc/i18n/formatters";
// Not: `${discount}%`
// Not: new Date().toLocaleDateString()
```

## Locale in URLs

- All public pages: `/ru/...`, `/en/...`, `/uk/...`
- The `locale` param is always the first dynamic segment
- Use navigation helpers from `apps/web/src/i18n/navigation.ts`:

```ts
import { Link, useRouter } from "@/i18n/navigation";
// Not: import Link from 'next/link'
```

## Locale Switching

Never redirect using `window.location` — use the locale-aware
`usePathname` + `useRouter` from next-intl to preserve the
current path while changing locale.
